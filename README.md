# Детекция дорожных знаков на изображениях и видео

## Описание проекта

Проект посвящён сравнению нескольких архитектур нейронных сетей для задачи детекции дорожных знаков на изображениях и видео. В рамках работы реализованы скрипты обучения, оценки и анализа результатов, а также веб-сервис на Django для запуска обученных моделей на пользовательских файлах.

## Цель работы

Цель работы состоит в сравнении качества, скорости и практической применимости разных моделей детекции объектов на задаче распознавания дорожных знаков.

## Используемые модели

- **YOLOv8n**  
  Лёгкая одностадийная модель детекции. Используется как базовый вариант для оценки баланса качества и скорости.

- **YOLOv11n**  
  Более современная модель семейства YOLO. Рассматривается как компактная архитектура для сравнения с базовой YOLOv8n.

- **YOLO26n**  
  Актуальная компактная модель YOLO-линейки. Используется для проверки качества новой архитектуры на выбранном датасете.

- **RT-DETR-l**  
  Transformer-based detector, ориентированный на высокое качество детекции. В проекте используется для сравнения с быстрыми YOLO-моделями.

- **Faster R-CNN ResNet50-FPN**  
  Классическая двухэтапная архитектура детекции объектов. Применяется как представитель более традиционного подхода к object detection.

## Датасет

Используется датасет **Traffic Signs Detection Europe** в формате YOLO. Датасет содержит 55 классов дорожных знаков.

- train: 3253 изображения
- valid: 1128 изображений
- формат разметки: `class_id x_center y_center width height`

Корректность структуры датасета и разметки проверялась скриптом `src/inspect_dataset.py`.

## Основные эксперименты

В проекте проведены эксперименты по обучению и сравнению моделей детекции дорожных знаков. Для оценки использовались метрики качества, скорость инференса и практические характеристики моделей.

Основные виды анализа:

- обучение моделей на одном датасете;
- benchmark скорости инференса;
- анализ ошибок на validation-выборке;
- анализ устойчивости latency и FPS;
- анализ влияния порога confidence;
- сравнение качества и скорости разных архитектур.

## Результаты

| Модель                    | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 | Latency, ms |     FPS |
| ------------------------- | --------: | -----: | ------: | -----------: | ----------: | ------: |
| YOLOv8n 640               |     0.721 |  0.731 |   0.769 |        0.684 |       7.128 | 140.282 |
| YOLOv11n 640              |     0.774 |  0.713 |   0.772 |        0.681 |       9.233 | 108.305 |
| YOLO26n 640               |     0.495 |  0.540 |   0.524 |        0.465 |       8.675 | 115.269 |
| YOLOv8n strong aug        |     0.709 |  0.686 |   0.740 |        0.631 |       7.104 | 140.768 |
| RT-DETR-l 512             |     0.862 |  0.862 |   0.905 |        0.803 |      16.033 |  62.371 |
| Faster R-CNN ResNet50-FPN |     0.797 |  0.954 |       — |            — |      20.495 |  48.792 |

RT-DETR-l показала лучшее качество по mAP. YOLOv8n продемонстрировала лучший баланс скорости и качества. Faster R-CNN показала высокий recall, но уступила по скорости. YOLO26n в текущих условиях показала худшие итоговые метрики.

Для Faster R-CNN mAP не указывается, так как модель оценивалась отдельно по TP/FP/FN, precision, recall и latency.

## Веб-сервис

Веб-сервис реализован на Django и предназначен для демонстрации обученных моделей на изображениях и видео.

Возможности сервиса:

- регистрация и авторизация пользователей;
- загрузка изображений и видео;
- выбор модели детекции;
- выбор лучших или последних весов модели;
- сохранение истории запусков;
- отображение статистики пользователя;
- вывод найденных объектов на русском языке.

## Запуск без Docker

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
cd service
python manage.py migrate
python manage.py seed_model_weights
python manage.py runserver
```

После запуска сайт доступен по адресу `http://127.0.0.1:8000/`.

## Запуск через Docker

```powershell
docker compose up --build
```

После сборки и запуска контейнера сайт доступен по адресу `http://127.0.0.1:8000/`.

Датасет подключается в контейнер как read-only volume. Веса моделей и результаты экспериментов используются из директории `outputs/`.

Если веса моделей не включаются в репозиторий из-за ограничений GitHub на размер файлов, их необходимо поместить в соответствующие директории внутри `outputs/runs/.../weights/` перед запуском сервиса.

## Smoke tests

Тестовые изображения и видео для быстрой проверки работы сервиса находятся в директории `smoke_tests/`.

## Вывод

Проект подтвердил различия между быстрыми YOLO-моделями, transformer-based RT-DETR и двухэтапной Faster R-CNN. RT-DETR-l показала лучшее качество по mAP, а YOLOv8n лучше подходит для быстрого прикладного режима. Веб-сервис демонстрирует практическое применение обученных моделей для обработки изображений и видео.
